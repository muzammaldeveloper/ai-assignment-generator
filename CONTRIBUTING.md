# Contributing to AI Assignment Generator

Thank you for considering a contribution.

## Development Setup

1. Fork and clone the repository.
2. Create a feature branch from main.
3. Set up backend and frontend locally.
4. Run tests and lint checks before opening a pull request.

Backend:

- Change directory to Backend
- Create virtual environment and activate it
- Install dependencies from requirements.txt
- Copy .env.example to .env and configure values

Frontend:

- Change directory to Frontend
- Install dependencies with npm ci
- Start development server

## Pull Request Guidelines

1. Keep pull requests focused and small.
2. Include tests for behavior changes.
3. Update docs when changing APIs, settings, or architecture.
4. Use clear commit messages and PR descriptions.

## Code Quality Checklist

- Backend lint passes
- Backend tests pass
- Frontend lint passes
- Frontend build succeeds
- No secrets committed

## Issue Reporting

When opening a bug report, include:

- Steps to reproduce
- Expected and actual behavior
- Logs or stack traces
- Environment details

## Contributor Conduct

By participating, you agree to maintain a respectful and inclusive environment.
